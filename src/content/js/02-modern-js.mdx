A tour of the most useful recent additions — the "edge JS" you'll see in modern codebases.

## Optional chaining `?.` (2020)

Short-circuits to `undefined` instead of throwing when something in the chain is null/undefined.

```js
const name = person?.name;          // no error if person is null
const city = person?.address?.city;
maybeFn?.();                        // call only if it exists
```

## Nullish coalescing `??` (2020)

A default **only** when the left side is `null`/`undefined` (unlike `||`, which also triggers on
`0`, `''`, `false`).

```js
console.log(user.age ?? 18);   // 18 only if age is null/undefined
```

## Logical assignment `&&=` `||=` `??=` (2021)

```js
a ||= 3;    // a = a || 3
a ??= 7;    // a = a ?? 7
```

## Dynamic imports & top-level await (2020 / 2024)

```js
const mod = await import('./module.js');   // load on demand
// In a module you can also: const data = await fetchData();
```

## Promise helpers

```js
// allSettled (2020) — wait for all, never short-circuits on rejection
const results = await Promise.allSettled([p1, p2]);

// any (2021) — first to fulfil wins; AggregateError if all reject
const first = await Promise.any([p1, p2]);

// withResolvers (2023) — promise + its resolve/reject in one go
const { promise, resolve, reject } = Promise.withResolvers();
```

## Strings & arrays

```js
'a+b+c'.replaceAll('+', ' ');     // 2021
[1, 2, 3].at(-1);                 // 3   (2022)

// Immutable array ops (2023) — return a copy, original untouched
[3, 1, 2].toSorted();             // [1, 2, 3]
[1, 2, 3].toReversed();           // [3, 2, 1]
[10, 20, 30].with(1, 99);         // [10, 99, 30]

// Group by a key (2023)
Object.groupBy(animals, (a) => a.type);
```

## Class fields (2022)

```js
class Counter {
  #count = 0;              // private field
  static label = 'demo';   // static field
  increment() { this.#count++; }
}
```

## New export syntax

```js
export const name = 'John';            // named
export default function add(a, b) {}   // default
export { name as myName } from './m.js'; // re-export + rename
```

<Callout type="do">`?.` and `??` together eliminate most defensive `if (x && x.y)` checks. Reach for the immutable array methods (`toSorted`, `with`) when you must not mutate shared data.</Callout>
