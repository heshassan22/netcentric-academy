## Variables & constants

Declare with `let` or `const` (avoid `var`). Assign, then optionally reassign.

```js
let name = 'John';       // declare + assign
name = 'Bob';            // ok

const id = 42;           // can't be reassigned
// id = 7;               // TypeError

const list = [];         // the binding is constant, the contents aren't
list.push('hello');      // ['hello']  ‚Üê fine
```

<Callout type="do">Default to `const`; use `let` only when you must reassign. Never `var`.</Callout>

## Scope

- **Global** ‚Äî accessible everywhere
- **Function** ‚Äî visible only inside the function
- **Block** ‚Äî `let`/`const` inside `{ }` (if, for, while) are visible only in that block

**Lexical scope:** an inner function can read variables from its enclosing scope.

```mermaid

      subgraph B["Block scope ‚Äî if / for / while"]

```

Inner scopes can read outward; outer scopes cannot read in.

```js
const name = 'John';
function outer() {
  const surname = 'Wick';
  // inner can see name + surname
  return `${name} ${surname}`;
}
// surname is NOT visible out here
```

## Classes & `this`

A **class** is a blueprint for creating objects. Each object you build from it is an **instance**.
Use a class when you have several things that share the same shape and behaviour ‚Äî like every
component on a page.

```js
class Greeter {
  // The constructor runs once, when you create an instance with `new`.
  // It sets up the instance's starting data (its "state").
  constructor(name) {
    this.name = name;        // store a property ON this instance
  }

  // A method ‚Äî a function that belongs to the class.
  greet() {
    console.log(`Hello ${this.name}`);
  }
}

const g = new Greeter('World'); // `new` builds an instance and runs the constructor
g.greet();                      // Hello World
```

Three words to be clear on:

- **`new`** ‚Äî creates a fresh, empty instance and runs the `constructor`.
- **property** (`this.name`) ‚Äî a piece of data stored on the instance. **method** (`greet`) ‚Äî a
  function the instance can run.
- **`this`** ‚Äî inside the class, `this` refers to **the specific instance you're working with**.
  Each instance has its own.

### `this` is per-instance

Make two instances and `this.name` is different in each ‚Äî the methods are shared, the data is not:

```js
const a = new Greeter('Ada');
const b = new Greeter('Linus');

a.greet(); // Hello Ada
b.greet(); // Hello Linus
```

<Callout type="warn">**The classic `this` gotcha.** When you pass a method as a callback, it can lose its `this`:

```js
button.addEventListener('click', g.greet);   // ‚ùå inside greet, `this` is no longer `g`
```

Fix it with an **arrow function** (arrows keep the surrounding `this`) or `.bind`:

```js
button.addEventListener('click', () => g.greet());  // ‚úÖ
button.addEventListener('click', g.greet.bind(g));  // ‚úÖ
```
</Callout>

<Callout type="note">This is exactly how the components in the next lessons work: each one is a **class**, and the loader creates an **instance** per element with `new MyComponent(element, options)` ‚Äî so `this.element` is *that* element, and every component keeps its own state.</Callout>
