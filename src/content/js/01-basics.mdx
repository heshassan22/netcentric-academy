## Variables & constants

Declare with `let` or `const` (avoid `var`). Assign, then optionally reassign.

```js
let name = 'John';       // declare + assign
name = 'Bob';            // ok

const id = 42;           // can't be reassigned
// id = 7;               // TypeError

const list = [];         // the binding is constant, the contents aren't
list.push('hello');      // ['hello']  ‚Üê fine
```

<Callout type="do">Default to `const`; use `let` only when you must reassign. Never `var`.</Callout>

## Scope

- **Global** ‚Äî accessible everywhere
- **Function** ‚Äî visible only inside the function
- **Block** ‚Äî `let`/`const` inside `{ }` (if, for, while) are visible only in that block

**Lexical scope:** an inner function can read variables from its enclosing scope.

```mermaid

      subgraph B["Block scope ‚Äî if / for / while"]

```

Inner scopes can read outward; outer scopes cannot read in.

```js
const name = 'John';
function outer() {
  const surname = 'Wick';
  // inner can see name + surname
  return `${name} ${surname}`;
}
// surname is NOT visible out here
```

## Data types

`String`, `Number`, `Boolean`, `Array`, and `Object` (functions, DOM nodes ‚Äî everything else ‚Äî are objects).

## Operators

```js
6 + 9;            // 15      (also string concat: 'a' + 'b')
20 / 2;           // 10
let n = 4;
n === 3;          // false   (strict equality)
n !== 3;          // true
```

## Control flow & loops

## Functions

```js
function sum(a, b) { return a + b; }
sum(3, 5); // 8
```

## Classes & `this`

A class is a template for objects (instances). `this` is the instance created with `new`.

```js
class Greeter {
  constructor(name) { this.name = name; }
  greet() { console.log(`Hello ${this.name}`); }
}
const g = new Greeter('World');
g.greet(); // Hello World
```

## Events (a first look)

button.addEventListener('click', () => console.log('clicked'));
