A component usually needs two things from the page: **configuration** (parameters) and access to
its **inner elements** (DOM references).

## Passing parameters

Pass data from HTML to JS with a `data-nc-params-<ComponentName>` attribute holding **valid
JSON**. The loader parses it and hands it to the constructor as `options`.

```html
<div data-nc="MyComponent"
     data-nc-params-MyComponent='{"text":"Hello from params"}'>
</div>
```

```js
class MyComponent {
  constructor(element, options) {
    this.element = element;
    this.options = options;          // { text: "Hello from params" }
    this.init();
  }
  init() {
    console.log(this.options.text);  // "Hello from params"
  }
}
```

<Callout type="warn">The params attribute must be **well-formed JSON** (double-quoted keys/strings). A trailing comma or single quotes will fail to parse.</Callout>

This is the bridge from the **back end / authoring** to your JS: anything HTL can output, you can
pass into the component.

## DOM references

`this.element` is the component's root node (the one with `data-nc`). Find inner elements
relative to it — **not** with document-wide queries.

```js
init() {
  this.title  = this.element.querySelector('.my-component__title');
  this.items  = this.element.querySelectorAll('.my-component__item');
}
```

<Callout type="do">Always scope queries to `this.element`. A page may contain many instances of the same component — `document.querySelector('.title')` would grab the wrong one.</Callout>

## 🛠 Practice

Read a `text` param and write it into `.my-component__text`:

<Reveal>

```html
<div data-nc="MyComponent"
     data-nc-params-MyComponent='{"text":"Hello My Component (from params)"}'>
  <span class="my-component__text"></span>
</div>
```

```js
init() {
  const el = this.element.querySelector('.my-component__text');
  if (el) el.innerHTML = this.options.text;
}
```

</Reveal>
