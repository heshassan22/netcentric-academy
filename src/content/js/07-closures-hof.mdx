Two ideas that power the functional style — and your component `init` pattern.

## Closure — a function remembers where it was created

A function keeps access to the variables in scope when it was defined, even after that outer function
has returned. That "remembered" state is **private** and survives between calls.

```js
const makeCounter = (start) => {
  let count = start;          // private to each counter
  return () => {
    count += 1;
    return count;
  };
};
```

Each call to `makeCounter` makes a **fresh** `count` — exactly how `init(root)` gives every component
its own state.

## Higher-order functions

A **higher-order function** takes a function as an argument, returns one, or both. You already use them:
`map`, `filter`, `addEventListener` all take a function. And `makeCounter` *returns* one.

```js
const onlyOnce = (fn) => {
  let done = false;
  return (...args) => {
    if (done) return;
    done = true;
    return fn(...args);
  };
};
```
