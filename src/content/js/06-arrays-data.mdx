Most JS work is reshaping arrays of data. A handful of methods cover almost everything — and they
**return new values** instead of mutating, which suits the functional style.

## The core methods

| Method | Returns | For |
|---|---|---|
| `map` | a new array, same length | transform each item |
| `filter` | a new, shorter array | keep items that pass a test |
| `reduce` | one value | fold a list into a total/object |
| `find` | the first match (or `undefined`) | look one up |
| `some` / `every` | a boolean | "any?" / "all?" |

They **chain** — each returns something the next can work on:

```js
const names = products
  .filter((p) => p.inStock)
  .map((p) => p.name);
```

## Keep it immutable

`map`/`filter` already return copies. For sorting, use `toSorted()` (not `sort()`, which mutates), and
spread to copy before changing:

```js
const cheapestFirst = [...products].sort((a, b) => a.price - b.price); // copy, then sort
```
