Pick up the components you built in the [mid-module checkpoint](checkpoint-assignment) and take them
to production quality — applying lessons **07–11**: shipping the JS, extracting modules, surviving
author re-render, and deferring heavy work. Budget **~2 hours**. Build it in `ui.frontend`.

<Callout type="note">Extend the same components — don't restart. If you skipped the checkpoint, do [Part 1](checkpoint-assignment) first (the Counter/Toggle/Tabs); everything here hardens those.</Callout>

## Part 1 — Modules & shipping (Intermediate, ~45min)

- [ ] Extract a piece of **pure logic** (a debounce, a formatter, the IntersectionObserver setup) into a reusable **module** under `modules/`
- [ ] Wire your components into the publish entry so the build bundles them (`export … from './components/…'`)
- [ ] Confirm the loader picks them up via `data-nc` — no manual instantiation

**Acceptance:** shared logic lives in a `modules/` file imported by more than one component; nothing is hand-wired outside the loader.

## Part 2 — Author mode & robustness (Advanced, ~45min)

- [ ] Make `init()` **idempotent** (guard against double-init) so it survives author re-rendering
- [ ] **Progressive enhancement**: the content works without JS; add a `<noscript>` fallback where needed
- [ ] A `destroy()` that removes listeners (clean teardown on re-render)

**Acceptance:** no duplicate listeners after a re-init; the page is usable with JS disabled; re-rendering a component in author mode doesn't leak handlers.

## Part 3 — Performance patterns (Advanced, ~30min)

- [ ] A **lazy-loading image/section** component using **IntersectionObserver** (reuse the module from Part 1)
- [ ] **Debounce** a `resize` (or `input`) handler; add `{ passive: true }` to a scroll listener
- [ ] `dynamic import()` a heavy dependency only when a component actually needs it

**Acceptance:** heavy work is deferred until visible; scroll/resize handlers don't jank; the heavy dependency isn't in the initial bundle.

## Stretch goals

- Extract pure logic into a tiny module and **unit-test** it (see the Unit Testing track).
- Add a custom-event "bus" so components coordinate without knowing about each other.
- Measure the bundle before/after `dynamic import()` and note the saving.

<Callout type="do">Done when: components are scoped, params-driven, coordinate via events, defer heavy work, survive re-init, and degrade gracefully without JS.</Callout>
