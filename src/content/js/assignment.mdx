Build **one real component — a Dropdown** — applying the track's patterns step by step. Each step
uses one lesson; do them in order. Build it in `ui.frontend`. Budget **~1.5 hours**.

<Callout type="note">**Starter project:** check out the `assignment/js-starter-dropdown` branch — the component (HTL, dialog, SCSS) and a demo page are all wired up; only `dropdown.clientlibs.js` is left as **TODOs**. Deploy it, then fill in that one file — no scaffolding. Stuck? Compare with the [answer](assignment-answer).</Callout>

<Callout type="note">Start from markup that already works without JS (a button + a list). Your JS only *enhances* it — that's progressive enhancement.</Callout>

The markup you enhance:

```html
<div class="dropdown" data-nc="Dropdown" data-nc-params-Dropdown='{"open": false}'>
  <button class="dropdown__toggle" aria-expanded="false">Menu</button>
  <ul class="dropdown__menu" hidden>
    <li><a href="#">One</a></li>
    <li><a href="#">Two</a></li>
  </ul>
</div>
```

## Step 1 — Register the class (lesson 03)

- [ ] A class taking `(element, options)` — store them, then call `init()`
- [ ] `register({ Dropdown })` so the loader creates one for every `data-nc="Dropdown"`

## Step 2 — Params & DOM refs (lesson 04)

- [ ] Read `open` from `this.options` (the parsed `data-nc-params`)
- [ ] Find the toggle button and the menu with `this.element.querySelector(…)` — scoped, never `document`

## Step 3 — Wire the event (lesson 05)

- [ ] On toggle click, show/hide `.dropdown__menu` (its `hidden` attribute) and keep `aria-expanded` in sync
- [ ] Close it when a click lands **outside** `this.element`

## Step 4 — Announce changes (lesson 06)

- [ ] Dispatch a `dropdown:toggle` custom event (with `{ open }` in `detail`) so other components can react

## Step 5 — Survive author mode (lesson 08)

- [ ] Guard `init()` so running it twice never binds a second listener (idempotent)

## Step 6 — Ship it (lesson 07)

- [ ] Export the class from the publish entry so the build bundles it and the loader runs it

<Callout type="do">Done when: the button toggles its own menu, two Dropdowns on a page work independently, it fires `dropdown:toggle`, and `init()` is safe to run twice. Best practice (lesson 10): scoped to `this.element`, no globals, works without JS.</Callout>

<Callout type="note">Stretch (lesson 09): debounce a `resize`/`input` handler if your component needs one. Prefer a real AEM component? See [AEM: Extend the Accordion](aem-assignment).</Callout>
