Build one component that ties the whole track together: a **Team Directory** that loads people from a
mock API, shapes the data, renders cards, and filters them live — all in the functional style. Build it
in `ui.frontend`. Budget **~2 hours**.

<Callout type="note">This combines the lessons — it is **not** a copy of any one demo. Data comes from the free **JSONPlaceholder** API: `https://jsonplaceholder.typicode.com/users`.</Callout>

The markup you enhance:

```html
<div class="directory" data-endpoint="https://jsonplaceholder.typicode.com/users">
  <input class="directory__search" type="search" placeholder="Search names…">
  <p class="directory__status"></p>
  <ul class="directory__list"></ul>
</div>
```

## Step 1 — Structure (lessons 03, 15)

- [ ] IIFE + a `SELECTORS` object; `init(root)` and `initAll()` over every `.directory`
- [ ] Read the API URL from `root.dataset.endpoint` (don't hard-code it)

## Step 2 — Fetch with states (lessons 10, 14)

- [ ] Show `Loading…` in the status; `fetch` the endpoint; check `res.ok` then `res.json()`
- [ ] On failure, show an error message in the status — never let it throw uncaught

## Step 3 — Shape the data (lessons 06, 08)

- [ ] In a **pure** helper, `map` each raw user to `{ name, email, company }` and **sort by name** (don't mutate the response)

## Step 4 — Render (lesson 11)

- [ ] Build the list with `createElement` + a `DocumentFragment`; use `textContent`, set `data-name` on each item; write once

## Step 5 — Filter live (lessons 05, 06)

- [ ] On the search `input`, filter the items by name (case-insensitive) and update the status to `showing X of Y`

## Stretch

- [ ] Debounce the search input (lesson 17)
- [ ] `AbortController` to cancel an in-flight request if it re-fetches (lesson 10)

<Callout type="do">Done when: it loads from the API, shows loading **and** error states, renders cards from the shaped data, filters live with a running count — and it's all functional (closures for state, pure transforms, queries scoped to `root`). Compare with the [answer](assignment-answer).</Callout>
