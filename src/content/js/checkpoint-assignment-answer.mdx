<Callout type="note">One correct solution — attempt the [checkpoint](checkpoint-assignment) first. The point is the functional pattern: `init` per element, state in a **closure** (no `this`), and clicks handled by **delegation** — all in plain browser JS.</Callout>

## The solution

```js
const SELECTORS = { root: '.tabs', list: '.tabs__list', tab: '.tabs__tab', panel: '.tabs__panel' };

const init = (root) => {
  const list   = root.querySelector(SELECTORS.list);
  const tabs   = root.querySelectorAll(SELECTORS.tab);
  const panels = root.querySelectorAll(SELECTORS.panel);
  if (!list) return;

  const select = (name) => {
    const panel = root.querySelector(`[data-panel="${name}"]`);
    if (!panel) return;
    panels.forEach(p => p.hidden = p !== panel);
    tabs.forEach(t => t.setAttribute('aria-selected', t.dataset.tab === name));
  };

  list.addEventListener('click', (event) => {
    const tab = event.target.closest(SELECTORS.tab);
    if (tab?.dataset.tab) select(tab.dataset.tab);
  });
};

const initAll = () => document.querySelectorAll(SELECTORS.root).forEach(init);

document.addEventListener('DOMContentLoaded', initAll);
```

## Why it's shaped this way

- **A closure per widget:** each `init(root)` call captures its own `list`/`tabs`/`panels`, so two tab groups on a page never clash — the closure does the job a class instance did (lesson 03).
- **No `this`:** `select` and the click handler are plain functions that **close over** local variables, so there's no `this` to lose — the lesson 02 gotcha simply doesn't apply here.
- **One delegated listener:** a single `click` on `.tabs__list`, not one per tab — read the target with `closest()` (lesson 05).
- **Modern JS:** NodeList `.forEach`, a template-literal selector, optional chaining (`tab?.dataset.tab`), and `setAttribute` coercing the boolean to `"true"`/`"false"` (lesson 04).

<Callout type="do">If your widget uses `init`/`initAll`, keeps state in the closure, and handles clicks with one delegated listener, you've nailed the functional pattern the project leans on — the AEM component lessons build straight on it.</Callout>
