Apply the JS module on a **real AEM component**. The annotated `Accordion` from lessons 06–12 is on a
branch — deploy it, read how it works, then **extend it**.

```bash
# in the `code` AEM project
git fetch && git checkout assignment/js-starter-accordion
mvn clean install -PautoInstallSinglePackage   # deploy to author :4502
```

Open `http://localhost:4502/content/academy/en/accordion-example.html` and read the annotated file:

```text
ui.frontend/src/main/frontend/academy/publish/components/accordion/accordion.clientlibs.js
```

**Your task:** it already *dispatches* an `accordion:toggle` custom event (`[06]`) that nothing listens
to yet. Make accordions **coordinate** — when a panel opens in one accordion, close any open panels in
the *other* accordions on the page, by listening to that event (not by reaching across components).

- [ ] Put a second Accordion on the page (author it, or add a node under `…/root/responsivegrid`)
- [ ] Listen for `accordion:toggle`; when it opens and didn't come from `this.element`, close your own panels
- [ ] Scope all DOM work to `this.element`; keep `init()` idempotent

<Callout type="do">Stretch: add keyboard support (Up/Down moves focus between headers), or make `allowMultiple` authorable via the dialog. The class is exported, so you can unit-test your logic (`[11]`).</Callout>
