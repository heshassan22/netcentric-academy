A component is edited in **author mode** and viewed in **publish**. The DOM and the needs differ
between them, and good AEM styling accounts for that.

## The decoration tag (the wrapper you didn't write)

When AEM renders a component on a page, it usually wraps it in a **decoration tag** — a `<div>` (by
default) carrying classes like the component's CSS class and edit-mode hooks. So your markup isn't
always the outermost element.

- Style your **own** root class (`.teaser`), not "the wrapper", so styles survive whether or not a
  wrapper is present.
- Control the wrapper per component via its `.content.xml`:

```xml
<!-- render no wrapper element -->
<jcr:root … cq:noDecoration="{Boolean}true"/>

<!-- or choose the wrapper tag -->
<jcr:root … cq:htmlTag="{nt:unstructured}" class="teaser-wrap"/>
```

<Callout type="note">The Style System adds the author-selected class to this **decoration tag** automatically (Components track). That's why your `--modifier` styles work without you touching the markup.</Callout>

## Author-only styling with `wcmmode`

Some styling is only for authors — outlines, "empty component" hints, edit affordances. Load it
behind a `wcmmode` guard so it never ships to publish. The project's `title` component does exactly
this:

```html
<!-- author-only clientlib -->
<sly data-sly-test="${wcmmode.edit}"
     data-sly-use.clientlib="${'/libs/granite/sightly/templates/clientlib.html'}"
     data-sly-call="${clientlib.all @ categories='academy.author.components.title'}"/>

<!-- publish styling, always loaded -->
<sly data-sly-use.clientlib="${'/libs/granite/sightly/templates/clientlib.html'}"
     data-sly-call="${clientlib.all @ categories='academy.publish.components.title'}"/>
```

The matching SCSS lives under `academy/author/...` so the build emits an `academy.author.*` category.

## Style the empty state

An author needs to see a component that has no content yet. Core Components render a **placeholder**
in edit mode; for custom components you can render a hint when content is missing:

```html
<sly data-sly-test="${!properties.text}"
     data-sly-use.tpl="core/wcm/components/commons/v1/templates.html"
     data-sly-call="${tpl.placeholder @ isEmpty=true}"/>
```

…and style edit-mode affordances with author CSS (only loaded in edit mode):

```scss
// academy/author/components/teaser/teaser.clientlibs.scss
.teaser:empty::before {
  content: "Teaser — add content";
  display: block;
  padding: 1rem;
  outline: 1px dashed currentColor;
}
```

## Edit-mode gotchas

<Callout type="warn">In author mode the component sits inside editor chrome (overlays, the decoration wrapper, the layout container). Avoid styles that assume your component is the outermost element or use `:first-child`/`:last-child` against the page — the wrapper and editor nodes will surprise you.</Callout>

<Callout type="do">Keep publish CSS free of edit-only concerns; put author affordances in an `academy.author.*` clientlib loaded only under `wcmmode.edit`. Publish pages stay lean, authors get the hints they need.</Callout>
