Pick up the SCSS you wrote in the [mid-module checkpoint](checkpoint-assignment) and turn it into a
**maintainable architecture** — partials, `@use`/`@forward`, token maps — then apply it the way you
would on a real AEM component. Applies lessons **05–09**. Budget **~2 hours**. Build it in your
project's `ui.frontend` SCSS.

<Callout type="note">Extend the same SCSS — don't restart. If you skipped the checkpoint, do [Part 1](checkpoint-assignment) first (variables, nesting, functions, mixins); everything here organizes and applies those.</Callout>

## Part 1 — Architecture (Intermediate)

- [ ] Split into **partials**: `_variables.scss`, `_mixins.scss`, component partials
- [ ] Wire them with **`@use` / `@forward`** (no `@import`) — namespaced where it helps
- [ ] Put your tokens in a **map** (e.g. `$spacing: (sm: .5rem, md: 1rem, lg: 2rem)`) and read with `map.get`
- [ ] Generate a spacing/utility set with **`@each`**
- [ ] A **`@mixin`** breakpoint helper driven by your breakpoints map

**Acceptance:** one entry point (`_index.scss`) `@forward`s the shared layer; components `@use` tokens instead of redefining them; no `@import`; changing a token ripples everywhere.

## Part 2 — Apply it to an AEM component (Advanced)

- [ ] Author **one component's styles as its own partial** (`hello.clientlibs.scss`-style), `@use`-ing the shared tokens/mixins — never redefining them
- [ ] Use **BEM-in-AEM** naming so the compiled selectors are flat, single-class, and namespaced to the component
- [ ] Add an **author-mode guard** for any edit-only styling (e.g. styles applied only under `.cq-Editable-dom` / behind a `wcmmode` wrapper)
- [ ] Sanity-check the **compiled output** is lean — no deep selector chains, no duplicated declarations

**Acceptance:** the component partial reuses the shared layer (no copy-pasted tokens); selectors stay flat BEM; author-only styling doesn't leak to publish.

## Stretch goals

- A typography mixin (`font-size` + `line-height` from a scale).
- Organise files ITCSS-style (settings → tools → components).
- Add a `map`-driven theme and switch it with `@each`.

<Callout type="do">Done when: a single change (a token, a mixin) ripples through the whole stylesheet, the file structure reads clearly, the component partial reuses the shared layer, and the compiled CSS is no bigger than your hand-written version.</Callout>
