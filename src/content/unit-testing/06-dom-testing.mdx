A lot of front-end code touches the **page** — reading elements, reacting to clicks. Jest runs your
tests in **jsdom**, a fake browser DOM, so you can build elements and click them **without opening a
real browser**.

## Turn on the DOM

Add this at the top of a DOM test file (or set it once in `jest.config.js`):

```js
/**
 * @jest-environment jsdom
 */
```

Now `document`, `querySelector`, events, etc. all work.

## Build some HTML, then check it

```js
test('shows the label', () => {
  document.body.innerHTML = `<button class="cta">Subscribe</button>`;
  const button = document.querySelector('.cta');
  expect(button).toHaveTextContent('Subscribe');
});
```

## Test a click

```js
test('toggles open on click', () => {
  document.body.innerHTML = `<button class="menu" aria-expanded="false">Menu</button>`;
  const button = document.querySelector('.menu');

  button.addEventListener('click', () => {
    const open = button.getAttribute('aria-expanded') === 'true';
    button.setAttribute('aria-expanded', String(!open));
  });

  button.click();                                          // simulate the click
  expect(button).toHaveAttribute('aria-expanded', 'true'); // check the result
});
```

## Test a component

Our components are `init(root)` functions: drop the markup on the page, call `init`, then check what
the user would see.

```js
import { init } from './counter.js';

test('counts up on click', () => {
  document.body.innerHTML = `
    <div class="counter">
      <button class="counter__inc">+</button>
      <output class="counter__value">0</output>
    </div>`;
  init(document.querySelector('.counter'));

  document.querySelector('.counter__inc').click();

  expect(document.querySelector('.counter__value')).toHaveTextContent('1');
});
```

<Callout type="do">Check what the **user sees** — visible text, attributes, what happens after a click — not the internal markup. Then a redesign of the HTML won't break a test whose behaviour is still correct.</Callout>

<Callout type="warn">jsdom is **not a real browser** — no layout or pixels, and some APIs return empty values (`getBoundingClientRect` gives zeros). For real visual/scroll behaviour use an end-to-end tool (Playwright/Cypress), not a unit test.</Callout>
