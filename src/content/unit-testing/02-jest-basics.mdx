**Jest** is the tool that finds your test files, runs them, and tells you which passed or failed. It
comes with everything you need built in — no extra setup to get started.

## A test file, line by line

```js
import { applyDiscount } from './cart.js';

describe('applyDiscount()', () => {        // a group of related tests
  test('takes 10% off', () => {            // one test
    expect(applyDiscount(200, 10)).toBe(180);   // the check
  });

  it('takes nothing off at 0%', () => {    // it() === test(), just reads nicer
    expect(applyDiscount(200, 0)).toBe(200);
  });
});
```

- **`describe('...', () => {})`** — a labelled box that groups tests for one thing.
- **`test('...', () => {})`** / **`it('...', () => {})`** — one test case (identical; pick whichever reads better).
- **`expect(value)`** — wraps the value so you can check it with a *matcher* like `toBe` (next lesson).

## Running them

Add a script to `package.json` so the whole team runs tests the same way:

```json
{ "scripts": { "test": "jest", "test:watch": "jest --watch" } }
```

Then:

```bash
npm test                    # run everything once
npm run test:watch          # re-run automatically as you edit
npm test -- cart.test.js    # just one file
npm test -- -t "10%"        # only tests whose name contains "10%"
```

## Setup & cleanup (lifecycle)

Sometimes every test needs the same starting point. These hooks run around your tests:

```js
let cart;

beforeEach(() => { cart = createCart(); });   // before EVERY test — fresh start
afterEach(() => { cart = null; });            // after every test — clean up

test('starts empty', () => {
  expect(cart.items).toHaveLength(0);
});
```

(`beforeAll` / `afterAll` also exist — they run **once** for the whole file.)

<Callout type="do">Put shared setup in **`beforeEach`** so each test starts clean and **independent** — a test must never depend on something a previous test left behind. Tests should pass in any order.</Callout>
