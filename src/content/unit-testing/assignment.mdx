Pick up the Jest project from the [mid-module checkpoint](checkpoint-assignment) and round it out —
test a component in the DOM, organize the suite, and make coverage **meaningful**. Applies lessons
**07–09**. Budget **~1.5 hours**.

<Callout type="note">Extend the same project — don't restart. If you skipped the checkpoint, do [Part 1](checkpoint-assignment) first (setup, matchers, mocking, async); everything here builds on it.</Callout>

## Get the starter

The starter is in the **kbc-fe** repo on branch **`unit-testing-assignment`** (it extends the
checkpoint project). `counter.js` is provided — write `counter.test.js` and tune coverage:

```bash
git clone https://github.com/heshassan22/kbc-fe.git
cd kbc-fe
git checkout unit-testing-assignment
npm install
npm test                # npm run test:coverage for the coverage report
```

## Part 1 — DOM testing

Given a `Counter` component class (takes `element`, increments on click):

- [ ] In **jsdom**, render markup, instantiate the component, **dispatch a click**, and assert the DOM updated (`toHaveTextContent`)
- [ ] Test a **custom-event** interaction (dispatch on `window`, assert a class/text changed)
- [ ] Use `@testing-library/jest-dom` matchers (`toHaveTextContent`, `toHaveAttribute`, `toBeVisible`)

**Acceptance:** the component test exercises real user behaviour (click → DOM change); it asserts on what the user sees, not internal fields.

## Part 2 — Organization & coverage

- [ ] Organize tests: co-locate `*.test.js`, group with nested `describe`, share setup with `beforeEach`/`afterEach`
- [ ] Run `npm run test:coverage` — the config already enforces an 80% threshold; write tests until it passes
- [ ] Find one **"vanity" test** (runs code, asserts nothing meaningful) and make it real

**Acceptance:** branch coverage is meaningful, not padded; the suite is grouped and readable.

## Stretch goals

- Parametrise edge cases with `test.each`.
- Break a line of production code on purpose — confirm a test goes red.
- Add a custom matcher for a domain assertion you repeat.

<Callout type="do">Done when: tests read as a spec, mocks isolate the unit, async is awaited, the component test mirrors real interaction, and coverage reflects **meaningful** assertions.</Callout>
