Lots of front-end code takes time — fetching data, timeouts, animations. The one rule: **the test
must wait** for that work to finish, or it ends *before* your check runs and passes by accident.

## Wait with `async` / `await`

```js
test('loads the user', async () => {       // note: async
  const user = await loadUser(1);          // wait for the promise
  expect(user.name).toBe('Ada');
});
```

<Callout type="warn">Forgetting `await` is the most common async-test mistake — the test finishes and is marked **passed** before the check ever runs. If a test "can never fail", you probably forgot to `await`.</Callout>

## Check a promise directly — `resolves` / `rejects`

```js
await expect(loadUser(1)).resolves.toMatchObject({ name: 'Ada' });  // it succeeds
```

## Don't wait for real time — fake the clock

Testing something on a timer (like a 300ms debounce)? Faking the clock lets you jump forward instantly
instead of really waiting:

```js
test('debounce runs once after 300ms', () => {
  jest.useFakeTimers();
  const fn = jest.fn();
  const debounced = debounce(fn, 300);

  debounced(); debounced(); debounced();   // called 3 times fast
  expect(fn).not.toHaveBeenCalled();        // nothing yet

  jest.advanceTimersByTime(300);            // jump 300ms forward
  expect(fn).toHaveBeenCalledTimes(1);      // ran exactly once

  jest.useRealTimers();
});
```

<Callout type="do">Use fake timers for anything time-based — debounce, throttle, polling, auto-dismiss. The test stays fast and gives the same result every run, instead of actually sleeping.</Callout>
