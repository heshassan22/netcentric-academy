## Part 1 — A DOM test (two events)

`counter.js` is a plain **functional** component (no framework, no dependencies). The test mounts
its markup, runs the component, and clicks the two buttons:

```js
// counter.test.js
import Counter from './counter.js';

function mount() {
  document.body.innerHTML = `
    <div class="counter">
      <button class="counter__inc">+</button>
      <button class="counter__reset">reset</button>
      <span class="counter__value">0</span>
    </div>`;
  const el = document.querySelector('.counter');
  Counter(el, { step: 1 });
  return el;
}

describe('Counter', () => {
  it('increments the display on click', () => {
    const el = mount();
    el.querySelector('.counter__inc').click();
    expect(el.querySelector('.counter__value')).toHaveTextContent('1');
  });

  it('resets to 0 when the reset button is clicked', () => {
    const el = mount();
    el.querySelector('.counter__inc').click();
    el.querySelector('.counter__reset').click();
    expect(el.querySelector('.counter__value')).toHaveTextContent('0');
  });
});
```

<Callout type="note">`toHaveTextContent` comes from `@testing-library/jest-dom`, wired up once via `setupFilesAfterEnv` in the Jest config.</Callout>

## Part 2 — Organization & coverage

The starter's `jest.config.cjs` already enforces the threshold — nothing to configure:

```js
coverageThreshold: {
  global: { branches: 80, functions: 80, lines: 80, statements: 80 },
},
```

```bash
npm run test:coverage    # fails the run if branches/lines drop below 80%
```

Just write tests until coverage passes, keeping each assertion meaningful.

<Callout type="do">Co-locate `*.test.js` next to the source, name tests as behaviour sentences, and gut-check coverage by breaking a line of production code — if no test goes red, that line isn't *meaningfully* covered.</Callout>
