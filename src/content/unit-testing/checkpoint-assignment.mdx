This is the **mid-module checkpoint** — apply lessons **01–06** by setting up Jest and testing pure
logic, mocks, and async code. **Keep your project**: the [final assignment](assignment) adds DOM
tests, organization, and coverage on top.

<Callout type="note">Test **behaviour**, not implementation. A good suite goes red when you introduce a real bug and survives a refactor that keeps behaviour the same.</Callout>

## Get the starter

The starter is in the **kbc-fe** repo on branch **`unit-testing-checkpoint-assignment`**. Clone it,
check out the branch, and work under `public/unit-testing/src/` — Jest is already configured:

```bash
git clone https://github.com/heshassan22/kbc-fe.git
cd kbc-fe
git checkout unit-testing-checkpoint-assignment
npm install
npm test          # confirm it runs and finds the *.test.js files
```

Each `*.test.js` has one worked example — add the rest to satisfy the parts below.

## Part A — Pure logic & matchers

Given `formatPrice(cents)`:

- [ ] Test the happy path with the **right matcher** (`toBe`)
- [ ] Cover **edge cases**: `0` (valid), a negative amount (throws), `null`/`undefined` (throws)
- [ ] Name tests as behaviour sentences; group with `describe`

**Acceptance:** happy-path + edge-case tests; matchers are specific (e.g. `toThrow('invalid amount')`).

## Part B — Mocking & async

Given `loadUser(id)` that calls `fetch`, and a `debounce(fn, ms)` helper:

- [ ] **Mock `fetch`** (`jest.fn().mockResolvedValue(...)`) and assert the URL + parsed result
- [ ] Test the **rejection** path with `await expect(...).rejects.toThrow(...)`
- [ ] Use **fake timers** to prove `debounce` fires **once** after the delay
- [ ] Reset mocks between tests (`afterEach(() => jest.resetAllMocks())`)

**Acceptance:** no real network; async assertions are awaited; the debounce test is deterministic (no real waiting).
