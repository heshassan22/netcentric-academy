The finish line: writing HTML that is **valid, conventional, compatible, and fast** — then putting
the whole track together in one document.

## Validate

Run your markup through the **[W3C validator](https://validator.w3.org/)**. It catches unclosed
tags, duplicate `id`s, misnested elements, and invalid attributes — bugs that cause subtle,
cross-browser rendering and accessibility problems.

## Coding conventions

- Lowercase tags and attributes; always quote attribute values.
- One UTF-8 encoding, declared first.
- Indent to show nesting; one block-level element per line.
- Meaningful `class` names (the project uses **BEM** — see the CSS track).
- Don't rely on the browser auto-fixing sloppy markup.

## Browser compatibility

- Check support on **[caniuse.com](https://caniuse.com/)** before using a newer feature.
- **Progressive enhancement**: start with HTML that works everywhere, then layer on CSS and JS.
  The content is readable even if CSS/JS fail.
- **Graceful degradation / fallbacks**: `<picture>` fallback `<img>`, `<video>` fallback text,
  `<noscript>` for no-JS.

## Performance basics

- `width`/`height` (or `aspect-ratio`) on media → no layout shift.
- `loading="lazy"` on off-screen images and iframes.
- `defer` your scripts; `preconnect`/`preload` critical resources.
- Keep the DOM lean — fewer, more meaningful elements.

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="utf-8">

  <meta property="og:image" content="https://acme.example/og.png">
  <link rel="icon" href="/favicon.svg" type="image/svg+xml">
  <link rel="stylesheet" href="/styles.css">
  <script src="/app.js" defer></script>
</head>
<body>
  <a class="skip-link" href="#main">Skip to content</a>

  <header>
    <a href="/"><img src="/logo.svg" alt="Acme Coffee" width="120" height="40"></a>

    <article>

      <p>Published <time datetime="2026-06-05">June 5, 2026</time>.</p>

      <section>
        <h2>What you need</h2>

          <li>Fresh beans</li>
