Forms collect user input. A good form is **labelled, validated, and submittable** — and most of
that is built into HTML.

## The form element

```html
<form action="/subscribe" method="post">
  …controls…
  <button type="submit">Subscribe</button>
</form>
```

- **`action`** — where the data goes. **`method`** — how:
  - **GET** appends data to the URL (search, bookmarkable, non-sensitive).
  - **POST** sends data in the request body (changes state, larger/sensitive data).
- For file uploads add `enctype="multipart/form-data"`.

## Labels — always

Every control needs a label, associated by `for` ↔ `id` (or by wrapping):

```html
<label for="email">Email</label>
<input type="email" id="email" name="email" required>

<!-- or wrap it -->
<label>Email <input type="email" name="email"></label>
```

<Callout type="warn">A `placeholder` is **not** a label — it disappears on typing and fails accessibility. Always provide a real `<label>`.</Callout>

## Common inputs

The `type` attribute changes the keyboard, validation, and UI:

```html
<input type="text">      <input type="email">    <input type="password">
<input type="number">    <input type="tel">      <input type="url">
<input type="date">      <input type="file">     <input type="range">
<input type="checkbox">  <input type="radio">    <input type="color">
<input type="hidden">
```

Plus the multi-line and choice controls:

```html
<textarea name="message" rows="4"></textarea>

<select name="country">
  <optgroup label="EU">
    <option value="be">Belgium</option>
    <option value="de">Germany</option>
  </optgroup>
</select>
```

## Grouping & built-in validation

```html
<fieldset>
  <legend>Account</legend>

  <label for="user">Username</label>
  <input id="user" name="user" required minlength="3">

  <label for="age">Age</label>
  <input id="age" name="age" type="number" min="18" max="120">

  <label for="site">Website</label>
  <input id="site" name="site" type="url" pattern="https://.*">
</fieldset>
```

- **`<fieldset>` + `<legend>`** group related controls (essential for radio/checkbox sets).
- **Validation attributes** — `required`, `minlength`/`maxlength`, `min`/`max`, `pattern`, and the
  `type` itself — let the browser validate before submit, **no JavaScript needed**.

<Callout type="do">Lean on native validation and input `type`s first. They give you mobile keyboards, date pickers, and error messages for free, and they work without JS.</Callout>

<Callout type="note">`<button>` defaults to `type="submit"` inside a form. Use `type="button"` for JS-only buttons and `type="reset"` to clear — being explicit avoids accidental submits.</Callout>
