<Callout type="do">Use `<strong>`/`<em>` for *meaning* (a screen reader emphasises them); use `<b>`/`<i>` only for styling with no added importance. In practice you'll rarely need `<b>`/`<i>`.</Callout>

## Links — the `<a>` element

```html
<a href="/docs/intro">Relative link (same site)</a>
<a href="https://example.com">Absolute link</a>
<a href="#section-2">Jump to a fragment (an id on the page)</a>
<a href="mailto:team@example.com">Email us</a>
<a href="tel:+3211223344">Call us</a>
<a href="/report.pdf" download>Download the PDF</a>
```

- **Relative** (`/docs`, `../img.png`) stays within your site; **absolute** (`https://…`) points
  anywhere.
- **Fragment** links (`#id`) scroll to the element with that `id` — the basis of in-page nav and
  "skip to content" links.

## Opening in a new tab — safely

```html
<a href="https://example.com" target="_blank" rel="noopener noreferrer">Opens in new tab</a>
```

<Callout type="warn">When you use `target="_blank"`, add `rel="noopener noreferrer"`. Without `noopener`, the new page can access `window.opener` and redirect your page (a phishing vector). Modern browsers do this by default now, but be explicit.</Callout>

<Callout type="note">Link text should make sense on its own — screen-reader users often jump link-to-link. Avoid "click here"; write "read the install guide".</Callout>
