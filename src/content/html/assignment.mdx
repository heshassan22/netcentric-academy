Pick up the **`recipe.html`** you started in the [mid-module checkpoint](checkpoint-assignment) and
take it to production quality — responsive media, an accessible form, and complete metadata —
applying everything from lessons **06–10**. Budget **~2 hours**.

<Callout type="note">Don't start from scratch — extend the same file. If you skipped the checkpoint, do [Part 1](checkpoint-assignment) first; everything here builds on that semantic skeleton. Keep validating with the [W3C validator](https://validator.w3.org/) as you go.</Callout>

## Part 1 — Media & embedded content (Intermediate)

- [ ] A hero image with a meaningful `alt`, explicit `width`/`height`, and a responsive `<picture>` (AVIF/WebP with a `<img>` JPG fallback)
- [ ] `loading="lazy"` on any below-the-fold image
- [ ] **(Optional)** an embedded `<iframe>` (e.g. a map or video) with a `title` and `loading="lazy"`

**Acceptance:** images are responsive, reserve their space (no layout shift), and every image has appropriate `alt` text.

## Part 2 — The review form (Advanced)

- [ ] A **"Leave a review" form**: labelled `name`, `email` (`type="email"`), a `rating` `<select>`, and a `comment` `<textarea>`
- [ ] Group the controls in a `<fieldset>` + `<legend>`
- [ ] Use **native validation** (`required`, input types, `minlength`) — no JS

**Acceptance:** every control has an associated `<label>`; the form validates natively; the fieldset/legend describe the group.

## Part 3 — Metadata, structured data & validation (Advanced)

- [ ] **Metadata** in `<head>`: `meta description`, Open Graph (`og:title`/`og:description`/`og:image`), a favicon, and a `<link rel="canonical">`
- [ ] **Structured data**: a JSON-LD `Recipe` block (`@type: Recipe`, name, author, ingredients, steps)
- [ ] **Performance**: `width`/`height` on all media, `defer` on any script
- [ ] **Validate** with the W3C validator and fix every error/warning

**Acceptance:** validator is clean; Open Graph + JSON-LD are present and well-formed.

## Stretch goals

- A **"skip to main content"** link as the first focusable element.
- A breadcrumb `<nav aria-label="Breadcrumb">`.
- A `<noscript>` fallback for anything you'd later enhance with JS.

<Callout type="do">Done when: it **validates clean**, works and reads correctly with CSS disabled, and you could hand the JSON-LD to a search engine. That's a production-quality HTML5 document.</Callout>
