Every element is a rectangular box. Getting layout right starts with understanding that box.

## The box model

From the inside out:

- **Content** ‚Äî the text/image area
- **Padding** ‚Äî space *inside* the border (transparent)
- **Border** ‚Äî around the padding
- **Margin** ‚Äî space *outside* the border (transparent)

![The CSS box model: a content box wrapped by padding, then border, then margin](img/css/box-model.svg)

```css
.box {
  width: 200px;     /* by default applies to the CONTENT box only */
  padding: 16px;
  border: 2px solid;
}
```

<Callout type="warn">By default `width`/`height` size only the **content** box, so padding + border are *added* on top ‚Äî a 200px box with 16px padding is actually 234px wide.</Callout>

<Callout type="do">Set `box-sizing: border-box` (globally is fine) so `width` includes padding and border. It's what most projects do:</Callout>

```css
*, *::before, *::after { box-sizing: border-box; }
```

Try it ‚Äî flip `box-sizing` between `content-box` and `border-box`, or change the padding/border, and watch the box's total size change:

## `position`

| Value | Behaviour |
|---|---|
| `static` | default; normal flow; ignores `top/left/‚Ä¶` |
| `relative` | offset *from its own* position; original space is kept |
| `absolute` | removed from flow; positioned to nearest *positioned* ancestor |
| `fixed` | removed from flow; positioned to the viewport; stays on scroll |
| `sticky` | flows normally, then "sticks" within its container on scroll |

```css
.parent  { position: relative; }     /* establishes a containing block */
.badge   { position: absolute; top: 0; right: 0; }
```

<Callout type="note">`absolute` looks for the nearest ancestor with a `position` other than `static`. If there is none, it anchors to the document body.</Callout>

## z-index & stacking contexts

The whole idea in one line: **`z-index` is local.** It only ranks elements **inside the same
stacking context** ‚Äî never across different ones.

A **stacking context** is a self-contained layer. A few things create one ‚Äî most commonly a
**positioned** element with a `z-index`, but also `opacity < 1`, a `transform`, a `filter`, or
`will-change`.

The catch: once an element makes a context, its children are **sealed inside it**. Their `z-index`
only competes with each other ‚Äî a child can't climb out, however big the number:

![Main (z-index 2) is painted over the whole Header (z-index 1) ‚Äî including its Menu (z-index 9999) ‚Äî because the Menu is trapped inside Header's stacking context](img/css/stacking-context.svg)

```css
.header       { position: relative; z-index: 1; }   /* a stacking context  */
.header .menu { position: absolute; z-index: 9999; } /* trapped inside it   */
.main         { position: relative; z-index: 2; }    /* a separate context  */
```

`.main` (2) beats `.header` (1), so it paints over the whole header ‚Äî and the menu's `9999` never
gets compared against `.main` at all.

<Callout type="warn">"My `z-index: 9999` doesn't work!" almost always means it's trapped in a parent stacking context. Fix the **parent's** stacking, not the child's number.</Callout>
