Two modern layout systems. Rule of thumb: **Flexbox for one dimension** (a row or a column),
**Grid for two dimensions** (rows *and* columns).

## Flexbox

A flex container distributes space among its items along a **main axis**, with alignment on the
**cross axis**.

```css
.toolbar {
  display: flex;
  flex-direction: row;        /* main axis = horizontal */
  justify-content: space-between;  /* along main axis   */
  align-items: center;             /* along cross axis  */
  gap: 1rem;
}
.toolbar .spacer { flex: 1; }       /* grow to fill      */
```

**Terminology:** flex container · flex items · flex-direction · main axis · cross axis.

### Play with every flex property

Toggle each property below and watch the boxes rearrange. The generated CSS updates live, so you
can connect each value to its effect — try `flex-direction: column` with different `align-items`, or
add `items` until `flex-wrap: wrap` kicks in.

<LayoutLab mode="flex" />

<Callout type="note">`align-items` aligns on the **cross axis** — so with `flex-direction: row` it moves items up/down, but with `column` it moves them left/right. Watch the boxes flip behaviour when you change direction. (`baseline` lines up the items' text baselines — that's why the boxes have different text sizes.)</Callout>

### A real-world example

Now the same properties on something concrete — change `justify-content`, `gap`, or `flex-direction`:

## CSS Grid

Grid lays out items into a 2-D grid of **tracks** (rows/columns) and **areas**.

```css
.layout {
  display: grid;
  grid-template-columns: 200px 1fr;   /* sidebar + fluid content */
  grid-template-rows: auto 1fr auto;
  gap: 1.5rem;
}
```

**Terminology:** grid container · grid items · grid line · grid cell · grid track · grid area.

### Play with grid layout

Pick a preset for `grid-template-columns` / `grid-template-rows` — or **type your own** value in the
field — and watch the boxes flow into the tracks. `justify-items`/`align-items` position each item
*inside* its cell; `justify-content`/`align-content` position the whole grid *within* the container;
`grid-auto-flow` controls the direction items are placed.

<LayoutLab mode="grid" />

<Callout type="note">Two things worth noticing as you experiment: `repeat(auto-fit, minmax(60px, 1fr))` reflows the column count on its own (no media query — the pattern behind the card grid below). And `justify-content`/`align-content` only have a visible effect when the tracks **don't fill** the container — try fixed columns like `repeat(3, 80px)` to see them move the whole grid.</Callout>

## New units, functions & keywords

| | Examples |
|---|---|
| **Units** | `fr` (fraction of free space) |
| **Functions** | `minmax(200px, 1fr)`, `repeat(3, 1fr)`, `fit-content(300px)` |
| **Sizing keywords** | `min-content`, `max-content` |

```css
.cards {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(240px, 1fr));
}
```

<Callout type="do">`repeat(auto-fit, minmax(…, 1fr))` gives you a responsive card grid with **no media queries** — it fits as many columns as will hold the minimum width, then stretches them.</Callout>

<Callout type="note">Use `gap` for spacing instead of margins on items — it works in both Flexbox and Grid and never adds an edge gap.</Callout>
