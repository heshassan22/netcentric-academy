Pick up the stylesheet you started in the [mid-module checkpoint](checkpoint-assignment) and turn
that foundation into a **responsive, themed UI** — applying lessons **05–09**, using only CSS.
Budget **~2 hours**. Try pieces in the lesson playgrounds first, then build it for real.

<Callout type="note">Extend the same stylesheet — don't restart. If you skipped the checkpoint, do [Part 1](checkpoint-assignment) first (reset, tokens, BEM); everything here builds on those foundations.</Callout>

## Part 1 — Layout (Intermediate)

- [ ] A **Flexbox** header/nav (logo left, links right, a `spacer` that grows)
- [ ] A **Grid** main area (content + sidebar) and a **card gallery** using `repeat(auto-fit, minmax(…, 1fr))`
- [ ] Use `gap` for spacing (never margins between items)
- [ ] Reserve media space with `aspect-ratio` (no layout shift)

**Acceptance:** the card gallery reflows responsively **without a media query**; header aligns with Flexbox; no CLS from images.

## Part 2 — Responsive, theming & modern CSS (Advanced)

- [ ] **Mobile-first** media queries that enhance upward, using a small set of breakpoints
- [ ] A **`prefers-reduced-motion`** query that disables non-essential animation, and (bonus) a `prefers-color-scheme` tweak
- [ ] A **light/dark theme** toggled by a class (`[data-theme="dark"]`) that only overrides custom properties
- [ ] **Transitions** on interactive elements (buttons/links) — animate `transform`/`opacity`, not layout
- [ ] One **modern-CSS** touch: `clamp()` fluid type, and `:has()` for a stateful style (e.g. a card with an image)

**Acceptance:** layout holds from 320px to wide desktop; dark theme is one class flip; motion respects `prefers-reduced-motion`.

## Stretch goals

- Use **cascade layers** (`@layer reset, components, utilities`) so overrides never need `!important`.
- Add a print-friendly tweak with `@media print`.
- Replace a "JS would do this" interaction with pure `:has()`.

<Callout type="do">Done when: the same markup looks polished and responsive from mobile to desktop, themes with a single class, and you didn't reach for `!important` once.</Callout>
